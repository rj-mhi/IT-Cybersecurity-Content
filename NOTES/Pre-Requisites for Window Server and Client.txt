Pre-Requisites for Window Server
-----------------------------------------------------
1. Install VM-Tools, VM-Settings, Network as Bridged

2. Using Server Manager, Disable Firewall, Assign IP, Rename Server

3. Installing Active Directory Role

4. Configure AD - Promoting to Domain Controller (DC)

5. Verify the AD and ping to client

6. Optional: Install Bginfo64.exe


Pre-Requisites for Window Client
---------------------------------------------------
1. Install VM-Tools, VM-Settings, Network as Bridged

2. Disable Firewall, Assign Ip, Rename Client

3. Optional - Install Bginfo64.exe

4. In Network Connections (ncpa.cpl) to properties, Add/Join client to Domain

5. Verify in Server Manager

Create AD users
----------------------	
1. Open Active Directory Users and Computers, and de-collapse the Domain Category, and find "Users" folder.

2. Select the "Users" folder and right click at the right side of the window and click "New" and then "User".

3. Assign a name and a password.

Delegating Permission to user
------------------------------------------
1. Open Active Directory Users and Computers (dsa.msc)

2. Right click the Domain Category and select Delegate Control

3. Select a user and click "Check Names" for a specified user.

4. Check the specified boxes for specified access and click "Next".

5. Then "Finish".

Joining a Client to a Domain
-----------------------------------------
1. In Client, as an Client Administrator, Open Settings by Windows Button + Page Break Key

2. Click Domain or Workgroup

3. Fill up the Domain box with your domain

4. Authenticate with the Domain Administrator's credentials (domain name and password)

5. Verify and Check in Windows Servers' Active Directory Users and Computers (dsa.msc)

Resigning/removing a Client in the Domain
--------------------------------------------------------------
1. Sign out from any Domain Users, Sign in the Client Administrator.

1. Click the Workgroup radio button to remove the client from the specified domain.

2. Authenticate with the Client Administrator.

3. Restart the PC.

4. Verify in the Windows Server's DSA.MSC.

How to Demote (De-assign) Active Directory Tools in the Server
-------------------------------------------------------------------------------------------

1. At the Server Manager, select Manage category.

2. Select Remove Roles and Features

3. Uncheck Active Directory Tools, then Error displays, and select "Demote this Domain Controller".

4. Select Force the Removal.

5. Proceed and Select Demote Button.

6. The Server will restart.

1. No GUI - smaller footprint (reduced resource usage).

2. Use less disk space, less RAM and CPU.

3. Fewer security vulnerabilities.

4. Reduced attack surface.

5. Fewer updates and reboots.

6. More stable than full GUI.

7. Ideal for production servers.

8. Best choice for Hyper-V.

9. Faster installation.

10. Lower maintenance effort.

11. Support all major sever roles.

12. PowerShell-based management.

13. Design for remote administration.
